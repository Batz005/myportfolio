College Enquiry Chatbot

Why and What? I choose to do this project during my 6th semester of college. Why? I observed during covid that students are often asking redundent questions to teachers or other staff. I heard teachers complaining about the same too. Since, they had to answer the same questions to multiple students and the students will also have to wait for the teachers to see their message and reply to it. Some of the students even started spamming teachers since they didn't receive answers on time. So, I decided to create a chatbot! I wanted this chatbot to answer all the general queries of the students. Teachers or other staff can give few details like the assignment deadlines, guidelines, slip test dates etc. This data is then used to answer the queries of the students.
This is a web application where students can login and ask the chatbot their queries. This chatbot will store all the details regarding the usual college enquiries including assignments, exams, deadlines, guidelines etc. Whenever a student has any query, they can ask the chatbot and the chatbot will answer their query within seconds. Thus, both students and the teacherâ€™s time will not be wasted on these general queries. Teachers will also be free from being spammed by students regarding the same issue. It saves time for the student as well as teaching and non-teaching staff. It saves time and manpower and reduces the work of the staff. A retrieval based chatbot is used in this scenario. The chatbot analyzes the userâ€™s query to extract the intents and entities in the query using natural language processing (NLP). Then the chatbot will select the most appropriate response based on similarity metrics. The chatbot also has a section where you can just chat with the chatbot casually (this part is implemented using chatterbot library). Students can use this web application through the front-end. It should consist of a login page and a user page where users can converse with the chatbot.
I want to include this chatbot to the student-centric social networking website. I will be working on this project in my final year. So, I wanted to make a few changes to the chatbot( make it a retrieval cum generative chatbot) and include it into the final year project. So, my final year project will ultimately combine everything and extend upon them.

Wallet Application using C++

Why and What? I choose to do this project during my 3rd semester of college. Why? I saw lot of students queuing and waiting for a lot of time near payment cell. I saw students pushing each other in crowds near the canteen to pay for their meals. So, I and my teammate decided to make a wallet application where we used a scanner to read the barcodes on our college id cards and used that as an id to login into the app and make payments. It's a bit like a credit/debit card. We used our college id cards as the id and students can scan their cards and change the money associated with that id no. to make payments. Students can use their cards at payment counters to recharge their cards. This will allow them to use the money that's been alloted to that id no. to make payments at available centres in college. The operator can retrieve the account details of the student if the operator's pc is connected to the main pc through windows homegroup. The details are stored in files (using the file read/write functionalities available in C). The main file contained all the mail ids' and the corresponding hashes using which the program identifies the filename of the account (which is created randomly during registration). This file contains the details regarding that particular user. (I didn't know how to use a database back then, so I used files to store my data as efficiently as I could) (This is how we envisioned it. Even though we completed the project, we didn't actually implement it in the college.)
What I learned from this project? This is my very first project. I didn't have proper guidance at the time. So, i wrote the entire code in a single file. I didn't use functions or classes as much as I should have. I didn't divide the code into different modules. So, whenever I faced an issue while working on the project, it was really hard to find the place where the issue occured. It was a nightmare. So, ever since this project. I started to modularize my code. I started dividing the project into small modules and stopped copy-pasting code and started using functions or modules to reuse the same code. I didn't know about databases when i worked on this project. So, I stored the data in files and used the function in c++ to access/modify files. At first, I just saved the account details of every customer in the same file. But then I observed that I'm reading through each and every line to find the account details of the current customer. That didn't seem like a good idea since I will read through the account details of every person from the start to the current customer's account details location. In terms of security and time required, it didn't seem like an efficient way. So, I divided everything. I created some kind of unique key for each customer, which will be available in the main file. So, whenever i needed to access a customer's details. I read through the main file, to find the correct login credentials and the corresponding key. This key is then used to create a filename, which will be the name of the file in which that corresponding customer's details are saved. When I later, learnt database management etc. I found out that the key I used back in my first project is like the primary key in a database. (Even though in a real database, the implementation might be lot more sophisticated.) This is the worst project I worked on (in terms of maintainability etc...) but it's also the project in which I learned a lot.
